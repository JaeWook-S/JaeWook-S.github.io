Project Overview

I participated in the RoboCup 2026 Humanoid Soccer League as a Team Leader, leading Inha-United, a joint team formed by four robotics labs at Inha University (SPARO, RCV, ARTEMIS, and RILS).

As a first-time RoboCup team, we had to build a collaborative robot system under a limited preparation period. I was responsible for Vision, system modularization, and the design of a Behavior Tree-based control architecture, focusing on improving the structural robustness and scalability of the robot software.

In addition, as the Team Leader, I managed the overall competition process, including the preparation of the Team Description Paper (TDP), team homepage, and GitHub repository, ensuring smooth participation from the qualification rounds through to the finals.

The team homepage and GitHub repository can be found here: Homepage | GitHub

Below, I describe the detailed technical approaches and preparation process for this project.

System Overview

Overall system architecture of the humanoid robot software pipeline.

Vision

The system processes RGB-D camera input to extract visual information and converts it into robot-relative perception data for localization and decision-making.

Localization

The localization module estimates the robot pose using odometry and a particle filter based on field landmarks.

Decision-Making

The decision-making module performs role assignment and behavior selection based on game state information from perception and localization modules.

Control

Robot motion control was performed using the Booster Robotics SDK, where pre-defined control policies were utilized to trigger corresponding low-level API commands for locomotion and actions.

Communication

UDP broadcast communication is used to share game state information from the game controller as well as robot pose, ball position, and pass signals among robots.
